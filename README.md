# Patch-Localization
分析了纯大模型4omin的结果，发现模型的幻觉很严重，如果同时需要给大模型描述，commit消息，代码diff并且让大模型判断结果会导致大模型遇到比较长的代码改动或者困难的改动时弱化思考能力，所以目前将单一任务分成三个角色，实现多智能体多角色完成漏洞补丁定位。
cve_analyst目前用于处理分析cve的描述内容
commit_analyst用于分析提交消息和代码改动，目前还没有调用设计的工具功能。
decision_maker用于整合上面两个智能体的回答，自己做出决策。
工具的设计目前还在修改，目前想法是展示用这个多智能体的版本在跑一次1669个cve，看一下准确率比如何
